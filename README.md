# math-expression-adt

A Java library implementing an immutable Abstract Data Type (ADT) for mathematical expressions, built around a recursive composite pattern.

## Overview

This project models algebraic expressions as an immutable tree structure. Each node in the tree is one of four types — a number literal, a named variable, an addition, or a multiplication — and every type is fully immutable and equal by structural value.

## File Guide

### Read these — they define the code

| File | Why it matters |
|---|---|
| `src/Expression.java` | The ADT contract. Defines the `Expression` interface and documents the four-variant type algebra. Start here. |
| `src/NumberExpr.java` | Leaf node for numeric literals. Shows the rep invariant, AF, `equals` with floating-point tolerance, and `hashCode` pattern. |
| `src/VariableExpr.java` | Leaf node for named variables. Shows rep validation via regex in `checkRep()`. |
| `src/AddExpr.java` | Internal node for n-ary addition. Shows defensive copy via `List.copyOf()` and recursive `toString`/`equals`. |
| `src/MultiplyExpr.java` | Internal node for n-ary multiplication. Structurally mirrors `AddExpr`. |
| `src/ExpressionTest.java` | JUnit test suite. Covers `toString`, equality, hash consistency, and order-sensitivity of `AddExpr`. |

## ADT Definition

```
Expression = Number(double value)
           + Variable(String name)
           + Add(List<Expression> terms)        -- requires >= 2 terms
           + Multiply(List<Expression> factors)  -- requires >= 2 factors
```

All four variants are immutable. There is no mutation after construction, no shared mutable state, and no rep exposure.

---

## Design Principles

- **Immutability** — `List.copyOf()` and `final` fields throughout; nothing can change after construction.
- **Rep safety** — `checkRep()` assertions guard invariants at construction time.
- **Structural equality** — `equals()` and `hashCode()` are value-based and consistent across all types. `NumberExpr.equals` uses a `1e-9` tolerance so `1.000` equals `1`.
- **Recursive structure** — `toString()`, `equals()`, and `hashCode()` all delegate to sub-expressions, leveraging the composite pattern naturally.

## Representation Invariants

| Class | Invariant |
|---|---|
| `NumberExpr` | Any valid `double` |
| `VariableExpr` | `name` matches `[a-zA-Z]+`, non-null |
| `AddExpr` | `terms.size() >= 2`, no nulls, stored as unmodifiable list |
| `MultiplyExpr` | `factors.size() >= 2`, no nulls, stored as unmodifiable list |

## Usage

```java
// 1 + x
Expression e = new AddExpr(List.of(new NumberExpr(1), new VariableExpr("x")));
System.out.println(e); // "1 + x"

// 2 * 3
Expression m = new MultiplyExpr(List.of(new NumberExpr(2), new NumberExpr(3)));
System.out.println(m); // "2 * 3"

// Equality is structural and order-sensitive
Expression a1 = new AddExpr(List.of(new NumberExpr(1), new VariableExpr("x")));
Expression a2 = new AddExpr(List.of(new VariableExpr("x"), new NumberExpr(1)));
a1.equals(a2); // false — order matters
```

## Running Tests

Uses **JUnit 4**, targets **Java 21**.

In Eclipse: right-click `ExpressionTest.java` → *Run As* → *JUnit Test*.

```bash
javac -cp .:junit-4.x.jar src/*.java
java  -cp .:junit-4.x.jar org.junit.runner.JUnitCore ExpressionTest
```

> **Known issue:** `ExpressionTest` mixes JUnit 4 (`org.junit.Assert`) with a JUnit 5-style `@Test` import and is missing the `public` class modifier. To fix: either make the class `public` and standardise on one JUnit version.
